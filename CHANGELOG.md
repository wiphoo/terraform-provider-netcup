# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## Policy

Per-release changelogs are **auto-generated by GoReleaser** from the commit
history and published on each [GitHub Release](https://github.com/wiphoo/terraform-provider-netcup/releases)
when a `vX.Y.Z` tag is pushed (see the "Release process" in
[CONTRIBUTING.md](CONTRIBUTING.md) and "Releasing" in [README.md](README.md)).

This file is the human-curated companion to those release notes:

- The `[Unreleased]` section below tracks user-facing changes as they land on
  `main`, so contributors and users can see what is coming before a tag is cut.
- On release, move the `[Unreleased]` entries under a new version heading and
  reset `[Unreleased]` to empty.
- Because commit messages follow the convention in
  [CONTRIBUTING.md](CONTRIBUTING.md), the generated release notes and this file
  should tell the same story.
